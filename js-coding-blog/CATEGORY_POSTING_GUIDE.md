# Category Posting Guide

이 문서는 JS Coding Blog에서 **카테고리(트랙)별 게시물 작성 방식**을 정리한 가이드입니다.

문제를 추가할 때는 단순히 난이도만 맞추는 것이 아니라,
**이 문제가 어떤 목적의 독자를 위한 것인지** 먼저 판단해야 합니다.

---

## 1. 오늘의 코테 (`today`)

### 목적
- 도전 욕구를 자극한다
- 매일 한 문제 풀고 싶은 흐름을 만든다
- 신선한 문제 경험을 제공한다

### 대상 독자
- 새로운 문제를 풀고 싶은 사람
- 짧고 재밌는 도전 요소를 원하는 사람
- 난이도별로 자기 수준을 확인하고 싶은 사람

### 게시물 톤
- 군더더기 없이 간결하게
- 문제 자체에 집중
- 설명은 명확하되 과하게 친절할 필요는 없음

### 권장 난이도 운영
- `easy`
- `medium`
- `hard`

### 권장 구조
```md
문제에 대한 한 줄 설명.

## 제한사항
- 입력 조건
- 반환 조건

## 예시
- 입력 / 출력 예시

## 힌트
- 선택적

## 해설
- 선택적
```

### 좋은 게시물 기준
- 제목만 봐도 도전 욕구가 생긴다
- 한 번쯤 생각할 포인트가 있다
- 너무 뻔하지 않다
- 짧아도 기억에 남는다

### 피해야 할 것
- 설명이 너무 길어서 문제 집중이 깨지는 경우
- 지나치게 튜토리얼처럼 쓰는 경우
- 학습 정보가 문제 자체를 압도하는 경우

## 2. 알고리즘별 코테 (`algorithm`)

### 목적
- 알고리즘 개념을 익히고 반복 연습하게 한다
- 유형별로 문제를 모아 실력을 쌓게 한다
- 초중급 학습자에게 성장 경로를 제공한다

### 대상 독자
- 알고리즘 개념이 아직 불안정한 사람
- 특정 유형을 묶어서 연습하고 싶은 사람
- 코테 감각을 단계적으로 키우고 싶은 사람

### 게시물 톤
- 학습 친화적
- 문제 의도가 분명해야 함
- 힌트와 해설의 교육적 가치가 중요함

### 대표 topic 예시
- `string`
- `array`
- `greedy`
- `hash`
- `stack-queue`
- `simulation`

### 권장 구조
```md
문제에 대한 한 줄 설명.

## 제한사항
- 입력 조건
- 반환 조건

## 예시
- 입력 / 출력 예시

## 힌트
- 어떤 방식으로 접근하면 좋은지

## 해설
- 왜 이 풀이가 맞는지
- 어떤 알고리즘 감각을 익히는 문제인지
```

### 좋은 게시물 기준
- 문제를 통해 특정 개념을 익힐 수 있다
- 힌트가 너무 정답 직전까지 가지 않는다
- 해설이 “왜 이 방식이 적절한지” 설명한다
- topic 분류가 분명하다

### 피해야 할 것
- 알고리즘 문제인데 단순 구현 문제처럼 끝나는 경우
- 해설이 정답 코드 소개에서 멈추는 경우
- 어떤 개념을 훈련하는지 모호한 경우

## 3. JS 메서드 학습 (`js-basic`)

### 목적
- JavaScript 메서드와 문법 감각을 익히게 한다
- 초보자의 진입장벽을 낮춘다
- 코테 입문 전에 언어 사용에 익숙해지게 한다

### 대상 독자
- JavaScript 초급자
- 메서드 사용이 아직 낯선 사람
- 알고리즘보다 먼저 언어 적응이 필요한 사람

### 게시물 톤
- 가장 친절해야 함
- 문제보다 학습 정보가 더 중요함
- 설명, 문법, 예시, 주의점이 반드시 필요함

### 대표 topic 예시
- `string-methods`
- `array-methods`
- `object-basics`
- `loop-function-basics`
- `high-order-functions`

### 권장 구조
```md
문제에 대한 한 줄 설명.

## 오늘의 메서드
- 오늘 배울 메서드 소개

## 메서드 설명
- 무엇을 하는 메서드인지

## 기본 문법
```js
arr.includes(value)
```

## 사용 예시
```js
['a', 'e'].includes('a') // true
```

## 주의할 점
- 대소문자 구분
- 원본 변경 여부
- 반환값 형태

## 제한사항
- 문제 조건

## 예시
- 입력 / 출력 예시

## 힌트
- 이 메서드를 어떻게 적용할지

## 해설
- 풀이 설명
- 왜 이 메서드가 적합한지
```

### 좋은 게시물 기준
- 메서드 개념이 처음인 사람도 이해 가능하다
- 문법과 예시가 짧고 명확하다
- 주의할 점이 실제 실수 포인트를 짚는다
- 문제 풀이가 메서드 학습과 자연스럽게 연결된다

### 피해야 할 것
- 그냥 일반 코테 문제처럼만 쓰는 경우
- 메서드 이름만 언급하고 설명이 없는 경우
- 예시 없이 문제만 주는 경우
- 해설이 “정답 코드”만 보여주고 학습 포인트를 설명하지 않는 경우

## 작성 전 체크 질문
문제를 쓰기 전에 아래를 확인합니다.

1. 이 문제는 어느 트랙에 가장 잘 맞는가?
2. 이 문제를 읽는 독자는 무엇을 얻어가야 하는가?
3. 설명 길이가 트랙 목적에 맞는가?
4. 힌트와 해설이 너무 과하거나 부족하지 않은가?
5. `js-basic`이라면 학습 정보가 충분한가?

---

## 한 줄 요약
- `today` → 문제 중심 / 도전 중심
- `algorithm` → 개념 중심 / 훈련 중심
- `js-basic` → 학습 중심 / 설명 중심

트랙별 목적이 다르기 때문에, 같은 형식으로 쓰지 않는 것이 중요합니다.
