# Topic Scope Guide

## 목적
다른 토픽의 맥락이 현재 토픽에 섞이는 문제를 줄인다.

## 규칙
1. 현재 메시지의 `topic_id`를 기본 스코프로 사용한다.
2. 명시적 요청이 없으면 다른 토픽의 맥락/할 일/결정을 가져오지 않는다.
3. 다른 토픽 내용을 쓰려면 먼저 사용자 확인을 받는다.
4. 메모리/히스토리 조회는 현재 `topic_id` 우선으로 필터링한다.
5. 지시 충돌 시 우선순위는 `현재 토픽 최신 지시 > 글로벌 메모리 > 타 토픽 히스토리`로 한다.
6. 답변 직전, 근거가 현재 토픽 기반인지 1회 점검한다. 불확실하면 확인 질문 후 진행한다.

## 확인 질문 템플릿
- "이 내용은 다른 토픽 문맥이 필요한데, 가져와도 될까요?"
- "현재 토픽 정보만으로 진행할까요, 아니면 이전 토픽도 참조할까요?"
